# ERC-721

L'[[ERC-721]] est le standard pour les jetons non fongibles sur Ethereum. Contrairement à l'[[ERC-20]] où chaque token est identique, les tokens ERC-721 représentent des actifs uniques avec des propriétés distinctes.

## Architecture du standard

Le standard ERC-721 définit des fonctions pour le transfert et la gestion des NFTs. Les fonctions include transferFrom, safeTransferFrom, approve et getApproved. Chaque token possède un tokenId unique qui le distingue.

L'événement Transfer est émis à chaque changement de propriétaire. L'événement Approval est émis lorsqu'un opérateur est autorisé pour un token spécifique.

## Identification des tokens

Chaque token ERC-721 possède un tokenId qui est un entier unique dans le contrat. Ce tokenId peut représenteranything from art digital à des objets de jeu ou des actifs du monde réel tokenisés.

Le tokenURI point vers les métadonnées off-chain qui décrivent les propriétés du token. Ces métadonnées peuvent inclure le nom, la description, l'image et autres attributs du actif.

## Applications dans l'écosystème

L'[[NFT ecosystem]] repose sur le standard ERC-721 pour la tokenisation des actifs uniques. Les marketplaces comme OpenSea permettent l'achat et la vente de NFTs ERC-721.

Les[[Trading bot]] qui opèrent sur les marchés NFT doivent comprendre les особенности de l'ERC-721. La liquidité et les volumes de trading diffèrent significativement des tokens fongibles.

## Trading algorithmique de NFTs

Le trading de NFTs ERC-721 présente des défis uniques. La nature unique de chaque token rend la comparison des prix plus complexe. Les[[Order book dynamics]] sur les marketplaces NFT fonctionnent différemment des carnets d'ordres sur les exchanges centralisés.

Les stratégies de[[Momentum trading]] peuvent être appliquées aux collections NFT avec des volumes足够的. Les métriques comme le prix plancher et le volume de ventes informent les décisions de trading.

## Considérations techniques

Le transfer sécurisé d'un ERC-721 requiert l'appel de safeTransferFrom qui vérifie que le destinataire est capable de recevoir des NFTs. Les contracts qui ne supportent pas l'interface ERC-721 peuvent perdre des tokens si mal utilisés.

Les[[Gas fees]] pour les transferts NFT sur[[Ethereum]] peuvent être élevées en période de congestion. Les solutions[[Layer 2]] comme[[Arbitrum]] offrent des alternatives moins coûteuses pour le trading NFT.

## Extensions et standards associés

ERC-721Metadata extend le standard avec des fonctions pour récupérer les métadonnées du token. ERC-721Enumerable ajoute des fonctions pour énumérer les tokens d'un propriétaire.

L'ERC-1155 permet de créer des tokens semi-fongibles qui combinent les caractéristiques des ERC-20 et ERC-721 dans un seul contrat.
